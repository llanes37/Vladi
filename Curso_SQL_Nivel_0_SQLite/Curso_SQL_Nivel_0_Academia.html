Prácticas del curso

Aquí tienes el recorrido completo del curso convertido en tarjetas de estudio: misión, ideas clave, script base y solución desplegable.

Práctica 01SELECT

Ver datos con SELECT

Leer sin tocar. Sacar columnas concretas y empezar a sentir comodidad con una tabla.

SELECT * FROM socios; SELECT nombre, ciudad FROM socios; SELECT titulo AS libro, genero AS tipo FROM libros;
Ver solución resumida
SELECT * FROM libros; SELECT nombre, ciudad FROM socios; SELECT titulo, paginas FROM libros; SELECT titulo AS libro, genero AS tipo FROM libros; SELECT prestamo_id, fecha_prestamo FROM prestamos;
Práctica 02WHERE

Buscar y filtrar

Texto con comillas, números con comparadores y vacíos con IS NULL.

SELECT nombre, ciudad FROM socios WHERE ciudad = 'Madrid';
Ver solución resumida
SELECT titulo FROM libros WHERE genero = 'Fantasia'; SELECT titulo, paginas FROM libros WHERE paginas > 200; SELECT nombre, telefono FROM socios WHERE telefono IS NULL; SELECT prestamo_id, devuelto FROM prestamos WHERE devuelto = 'NO'; SELECT nombre FROM socios WHERE nombre LIKE 'A%';
Práctica 03ORDER BY

Ordenar y limitar

Aprendes a pedir primero lo más relevante y a crear tops de resultados.

SELECT titulo, paginas FROM libros ORDER BY paginas DESC LIMIT 3;
Ver solución resumida
SELECT nombre, ciudad FROM socios ORDER BY nombre ASC; SELECT titulo, paginas FROM libros ORDER BY paginas ASC; SELECT titulo, paginas FROM libros ORDER BY paginas DESC LIMIT 3; SELECT prestamo_id, fecha_prestamo FROM prestamos ORDER BY fecha_prestamo DESC;
Práctica 04CREATE + INSERT

Crear tablas e insertar

Dejas de ser solo lector de datos y pasas a crear estructuras nuevas tú mismo.

CREATE TABLE IF NOT EXISTS salas_estudio ( sala_id INTEGER PRIMARY KEY, nombre TEXT NOT NULL, plazas INTEGER NOT NULL );
Ver solución resumida
DROP TABLE IF EXISTS talleres; DROP TABLE IF EXISTS salas_estudio; CREATE TABLE IF NOT EXISTS salas_estudio (...); INSERT INTO salas_estudio (...) VALUES (1, 'Sala Norte', 12); INSERT INTO salas_estudio (...) VALUES (2, 'Sala Centro', 8), (3, 'Sala Silencio', 20); CREATE TABLE IF NOT EXISTS talleres (...); INSERT INTO talleres (...) VALUES (1, 'Lectura rapida', 2), (2, 'Club de misterio', 1);
Práctica 05JOIN

Relacionar tablas

El salto importante: dejar de ver ids sueltos y ver información humana en una única salida.

SELECT p.prestamo_id, s.nombre, l.titulo FROM prestamos p JOIN socios s ON p.socio_id = s.socio_id JOIN libros l ON p.libro_id = l.libro_id;
Ver solución resumida
SELECT p.prestamo_id, s.nombre, p.fecha_prestamo FROM prestamos p JOIN socios s ON p.socio_id = s.socio_id; SELECT s.nombre, l.titulo, p.fecha_prestamo FROM prestamos p JOIN socios s ON p.socio_id = s.socio_id JOIN libros l ON p.libro_id = l.libro_id WHERE p.devuelto = 'NO';
Práctica 06GROUP BY

Contar y agrupar

Pasas de mirar filas sueltas a resumir el dataset con agrupaciones y conteos.

SELECT genero, COUNT(*) AS total_libros FROM libros GROUP BY genero;
Ver solución resumida
SELECT COUNT(*) AS total_libros FROM libros; SELECT COUNT(*) AS total_prestamos FROM prestamos; SELECT genero, COUNT(*) AS total_libros FROM libros GROUP BY genero; SELECT devuelto, COUNT(*) AS total_prestamos FROM prestamos GROUP BY devuelto; SELECT s.nombre, COUNT(*) AS total_prestamos FROM prestamos p JOIN socios s ON p.socio_id = s.socio_id GROUP BY s.nombre ORDER BY total_prestamos DESC;

Mini proyecto final

El cierre del nivel 0 mezcla todo lo anterior: leer, filtrar, ordenar, unir, contar y crear una tabla propia.

scripts/99_miniproyecto_final.sql

Objetivos de entrega

  • Mostrar libros con título y género.
  • Sacar socios de Madrid y socios sin teléfono.
  • Calcular top 2 libros más largos.
  • Mostrar préstamos no devueltos con nombre y título.
  • Contar socios y libros por género.
  • Crear la tabla ordenadores e insertar 3 filas.

Solución resumida

SELECT titulo, genero FROM libros; SELECT nombre, ciudad FROM socios WHERE ciudad = 'Madrid'; SELECT titulo, paginas FROM libros ORDER BY paginas DESC LIMIT 2; SELECT nombre, telefono FROM socios WHERE telefono IS NULL; SELECT p.prestamo_id, s.nombre, l.titulo FROM prestamos p JOIN socios s ON p.socio_id = s.socio_id JOIN libros l ON p.libro_id = l.libro_id WHERE p.devuelto = 'NO'; SELECT genero, COUNT(*) AS total_libros FROM libros GROUP BY genero; CREATE TABLE IF NOT EXISTS ordenadores ( ordenador_id INTEGER PRIMARY KEY, aula TEXT NOT NULL, operativo TEXT NOT NULL );

Errores típicos y arreglo rápido

Estos son exactamente los errores que más bloquean al empezar. Tenerlos visibles ahorra mucho tiempo.

"no such table"

Normalmente significa que no has ejecutado el script base de reset.

Ejecuta: scripts/00_reset_biblioteca_nivel0.sql

"no such column"

Revisa el nombre de la columna. SQL no adivina si escribes mal fecha_prestamo o telefono.

= NULL

Eso no funciona. Para vacíos reales se usa IS NULL.

SELECT nombre FROM socios WHERE telefono IS NULL;

Texto sin comillas

Para valores de texto necesitas comillas simples.

WHERE ciudad = 'Madrid'

No salen filas

No siempre es error. Puede que tu filtro no coincida con los datos del dataset.

UNIQUE constraint failed

En prácticas de creación e inserción suele significar que repetiste una clave primaria ya usada.
